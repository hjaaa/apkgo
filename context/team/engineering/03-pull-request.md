# Pull Request 规约

## 前言

PR 描述是评审的入口。统一的描述结构能让评审者（以及参与开发的 agent）快速定位变更意图、影响范围和验证情况，避免每个 PR 各写各的、关键信息靠追问补齐。

本规约定义团队统一的 PR 模板与填写要求，适用于所有仓库。

## 1 模板文件

- 1.1 【强制】 各仓库必须提供 PR 模板，路径固定为 `.github/PULL_REQUEST_TEMPLATE.md`

  该路径是 GitHub 的约定位置：网页创建 PR 与 `gh pr create` 交互式创建时会自动应用模板。

- 1.2 【强制】 模板内容从本规约「3 标准模板」原样复制；唯一允许的本地化是在「影响范围」的注释中补充本仓库的模块示例

  规则变更只发生在本规约中，各仓库的模板文件是本规约的机械复制，不得自行增删章节。

- 1.3 【强制】 通过 CLI / API 直接传入描述创建 PR（如 `gh pr create --body`，含 agent 提交的 PR）时，该方式不会自动应用模板，描述必须主动按模板结构组织

## 2 填写要求

- 2.1 【强制】 「变更说明」「影响范围」「变更类型」「验证方式」四节必填；其余各节不适用时勾选或注明「不涉及 / 不适用」，不得整节删除

- 2.2 【强制】 涉及对外接口、配置项或数据库变更时，必须说明兼容性、回滚方式和迁移影响

- 2.3 【强制】 PR 描述使用简体中文

- 2.4 【推荐】 「验证方式」附上可复现的证据：已执行的命令、curl/Postman 片段或截图

- 2.5 【推荐】 无关联 Issue 时删除 `Closes #` 行，避免误关联

## 3 标准模板

```markdown
## 变更说明

<!-- 简要说明本 PR 做了什么、为什么需要这个变更 -->

## 影响范围

<!-- 列出受影响的模块或目录 -->

## 变更类型

- [ ] Bug 修复
- [ ] 新功能
- [ ] 重构，不改变外部行为
- [ ] 配置 / 构建 / CI 变更
- [ ] 文档变更
- [ ] 其他：

## 验证方式

<!-- 填写已执行的命令、接口调用、截图或 Postman/curl 片段 -->

- [ ] 已运行单元测试 / 集成测试
- [ ] 已本地构建通过
- [ ] 已验证相关接口或页面
- [ ] 不适用，原因：

## 接口 / 配置 / 数据库变更

- [ ] 修改对外接口契约
- [ ] 修改配置项
- [ ] 包含数据库 schema 或种子数据变更
- [ ] 不涉及

如涉及，请说明兼容性、回滚方式和迁移影响：

## 关联 Issue

Closes #

## 给 Reviewer 的说明

<!-- 标出重点 review 区域、已知风险、需要特别关注的地方 -->
```
