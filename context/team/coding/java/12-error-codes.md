# 错误码（统一响应码）

本项目不设业务错误码体系。统一响应封装 `com.aiwork.common.core.util.R` 的 `code` 仅作**成功/失败标记**（int 型，0 成功、1 失败，见 `CommonConstants`），不承载错误分类与溯源信息；出错原因由 `msg` 承载。业务异常统一抛出并由全局异常处理器（`GlobalBizExceptionHandler`）转为 `R` 错误响应，禁止在业务代码中自造字符串错误码。未来若出现"仅靠 msg 无法定位问题"的真实场景，需要为 `R` 扩展业务错误码字段时，必须先做架构评审，禁止在单个模块内私自扩展。

1. 【强制】响应码不能直接输出给用户作为提示信息使用。

    **说明**：用户看到的是 `msg` 中的文案；堆栈（stack_trace）、排障信息与用户提示是互相关联但各司其职的整体，请勿互相越俎代庖。

2. 【强制】`msg` 是**用户可见文案**（前端拦截器直接弹出展示），要求简短清晰、提示友好；排障所需的业务信息（入参、上下文、堆栈）只记录在服务端日志中，禁止塞进 `msg`，也不要包含敏感数据。
